**Path**: `.cursor/rules/project-bootstrap.mdc`

- Follow Google JavaScript/TypeScript Style Guide conventions
- Enforce Test-Driven Development: write test first, then implement, then refactor
- Auto-research: before implementing, search for best practices; after each iteration, reflect and log learnings

#### 2b. Update Logging Rule (alwaysApply)

**Path**: `.cursor/rules/update-logging.mdc`

- **alwaysApply: true**
- At the end of each session/task, append to `UPDATE_LOG/YYYY-MM-DD_summary.md`
- Template: timestamp, author (LLM or Human), summary of changes, files modified
- Sort convention: one file per day with a summary in the filename, and rename it if the day's scope changes
- Auto-generate `UPDATE_LOG/README.md` as an index that links to all log entries

#### 2c. TDD Workflow Rule

**Path**: `.cursor/rules/tdd-workflow.mdc`

- **globs**: `**/*.test.{js,ts}`, `**/*.spec.{js,ts}`, `src/**/*.{js,ts}`
- Red-Green-Refactor cycle
- Write failing test first, implement minimum code to pass, then refactor
- Every feature/bugfix must have a corresponding test

#### 2d. Google Style Rule

**Path**: `.cursor/rules/google-style.mdc`

- **globs**: `**/*.{js,ts,jsx,tsx}`
- Google JavaScript Style Guide key conventions
- Naming: camelCase for vars/functions, PascalCase for classes/types, UPPER_SNAKE for constants
- JSDoc for public APIs
- Prefer `const` over `let`, never `var`
- 2-space indentation, semicolons required, single quotes
