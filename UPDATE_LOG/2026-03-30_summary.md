## Transcript NDJSON streaming (2026-03-30)

- **Why it felt slow**: `/api/transcript` still waits on YouTube timedtext + server parse before any bytes; the hard cap “under 5s” depends on network and caption size. The expensive part for the **browser** was one huge `JSON.parse` on the full cue array.
- **Change**: `POST /api/transcript?stream=1` returns **`application/x-ndjson`**: a `head` line (language, source, total), then **one JSON line per 100 cues** (`TRANSCRIPT_NDJSON_CHUNK_SIZE`, [`src/lib/render-model.js`](../src/lib/render-model.js)), then `done`. The client ([`src/ui/client.js`](../src/ui/client.js)) parses **line-by-line**, appends cues, and re-renders on rAF; status shows `{loaded} / {total} cues loaded` while streaming. Non-stream `POST /api/transcript` (no query) is unchanged for small payloads or debugging.
- **Profiling**: `?workspacePerf=1` logs `transcript_first_chunk_ms` and `transcript_stream_total_ms`.
- **Further gains (not implemented)**: cache watch-page fetch between workspace + transcript, stream-parse YouTube json3, or edge cache by `videoId`.

## Split workspace vs transcript API (2026-03-30)

- **`POST /api/workspace`** ([`src/worker.js`](../src/worker.js), [`src/lib/youtube.js`](../src/lib/youtube.js)): Returns video metadata and a **pending** transcript stub (caption language hint, `entries: []`). Small JSON, fast `JSON.parse`.
- **`POST /api/transcript`** (same): Same `{ url }` body; loads timedtext cues via [`fetchTranscriptPayload`](../src/lib/youtube.js). Client merges into `state.workspace.transcript` and clears `pending` ([`src/ui/client.js`](../src/ui/client.js)). **Trade-off:** the worker may fetch the YouTube watch page twice per load (once per endpoint); acceptable for separating payload latency.
- **Model** ([`src/lib/render-model.js`](../src/lib/render-model.js)): `makeWorkspacePayload` / `makeTranscriptApiPayload` and `transcript.pending` flag.

- **What**: [`parseApiJsonResponse`](../src/ui/client.js) replaces raw `response.json()` on workspace load so HTML bodies (wrong host / SPA fallback) surface [`statusApiHtmlInsteadOfJson`](../src/ui/client.js) instead of a cryptic parse error; [docs/cloudflare-github-deploy.md](../docs/cloudflare-github-deploy.md) troubleshooting section.
